# Planning

実装前に、要求 → 要件 → 設計の順でプランを固めるスキル。
正本は `.plans/<slug>/plan.md` のみとする。

## トリガー

- `/pj-planning`
- 「プランを立てたい」「要件を整理したい」「設計を詰めたい」
- 「pj-planning スキル」「実装前に計画したい」

## 起動時の分岐

`/pj-planning` 起動後、次のいずれかをユーザーに確認する。

1. **新規** — これからプランを作る
2. **既存を開く** — `.plans/` 配下の `plan.md` から選んで読み込む
   - `状態: draft` なら検討フェーズを継続
   - `状態: ready` なら実装フェーズへ案内

候補が複数あるときは勝手に選ばず、番号または slug で指定してもらう。

### 既存プラン選択テンプレート

```text
プランを開きます。どれを選びますか？
1. <slug1>/plan.md
2. <slug2>/plan.md
番号、slug、またはファイルパスで指定してください。
```

## 実行フロー

### フェーズ 1: 要求ヒアリング

新規プラン作成時、**検討に入る前に** 要求を整理する。
一度に聞く項目は 3〜5 個まで。足りなければ追加で聞く。

#### ヒアリング項目

1. **主題** — 何についてプランを立てるか（一言）
2. **背景** — なぜ今やるか、何が困っているか
3. **ゴール** — 完了したときにどうなっていればよいか
4. **スコープの目安** — 対象・対象外の初期イメージ
5. **制約** — 期限、技術制約、触ってはいけない領域（わかる範囲で）

#### ヒアリング後

1. slug を自動生成する（小文字・数字・ハイフンのみ）
2. `.plans/<slug>/plan.md` を **検討に入る前に** 作成する
3. [assets/plan.md](assets/plan.md) をベースに、ヒアリング内容を「概要」「背景・課題」「ゴール / 成功条件」「スコープ」へ反映する
4. 作成日を `YYYY-MM-DD` で記入する
5. プロジェクト名を補完する（未確定なら `unknown`。わかっていれば作成直後に埋める）
6. ファイルの存在と必須項目を確認する

同名 slug が既にある場合は別 slug を提案するかユーザーに確認する。

### フェーズ 2: 検討（grill）

1. `pj-grill-me` スキルの手順に従う
2. 合意が得られた内容は、その都度 `plan.md` の該当セクションへ追記する
   - 要件 → 「要件」
   - 設計判断 → 「設計」「決定ログ」
   - 未解決 → 「未確定事項」

#### grill の観点（順不同・必要なものだけ）

- スコープの境界（対象外の明示）
- 要件の受け入れ条件
- 既存システムとの整合（責務境界・データ・API）
- 設計の主要分岐（代替案と選定理由）
- 失敗モード・ロールバック
- テスト・検証方針

### フェーズ 3: ready 昇格

次をすべて満たし、ユーザーが「着手できる」「ready にして」と言ったときだけ昇格する。

- 「概要」が記入済み
- 「要件」が記入済み
- 「設計」が記入済み（意図的に設計不要と合意した場合は「設計不要」と明記）
- 「次のアクション」が記入済み
- 「未確定事項」が空、または着手をブロックしないと明記済み

昇格時は `plan.md` の `状態:` を `ready` に更新する。

### フェーズ 4: 実装への接続

- `ready` のプランを読み、次のアクションから着手を案内する
- 実装中にプランが変わったら `plan.md` を更新する

### draft への差し戻し

ユーザーが「draft に戻して」と言ったとき、`状態:` を `draft` に戻す。

## 既存プランを開いたとき

`plan.md` を読み込み、次の 4 点を簡潔に共有する。

- 概要
- 次のアクション
- 未確定事項
- 注意事項（決定事項・詰まりやすい点）

本文の長文転記は避ける。

- `draft` ならフェーズ 2（検討）を継続する
- `ready` ならフェーズ 4（実装への接続）へ進む

## ルール

### 正本

- 記録の正本は `.plans/<slug>/plan.md` のみ

### slug

- 使える文字は小文字・数字・ハイフンのみ
- `plan.md` の H1 に使う

### 必須セクション（新規プラン）

- プロジェクト名 / 作成日 / 状態 / 概要 / 背景・課題 / ゴール / スコープ / 要件 / 設計 / 未確定事項 / 次のアクション

### 任意セクション

- リスク・懸念 / 決定ログ / メモ

### バリデーション

- `.plans/` がなければ作成する
- プラン作成後にファイルの存在と必須項目を確認する

## テンプレート

新規プランは [assets/plan.md](assets/plan.md) をコピーして使う。
